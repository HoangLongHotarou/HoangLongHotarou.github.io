# Implementation Plan: Responsive Portfolio UI

**Branch**: `001-responsive-portfolio` | **Date**: 2026-03-14 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-responsive-portfolio/spec.md`

## Summary

Redesign the portfolio's CSS and component structure to be fully responsive across mobile (375px) and desktop (1440px) viewports. The implementation uses mobile-first CSS with CSS Grid for skill cards, Flexbox for header/blade, and moves the theme toggle in-flow. No external UI library is introduced.

## Technical Context

**Language/Version**: TypeScript 5, React 19, Vite 8  
**Primary Dependencies**: react-router-dom v7, CSS custom properties (no UI library)  
**Storage**: N/A (static SPA)  
**Testing**: Manual viewport testing via browser DevTools  
**Target Platform**: Web browser — latest 2 versions Chrome, Firefox, Safari, Edge  
**Project Type**: web-app (SPA, static)  
**Performance Goals**: FCP < 2 s on 4G; CLS < 0.1  
**Constraints**: No external CSS framework; must support existing light/dark theme tokens; mobile viewport minimum 375px  
**Scale/Scope**: Single-page portfolio, 2 routes (Home, System Design)
